# 🎛️ Pico RGB Keypad Playground

A collection of fun, beginner-friendly MicroPython projects for the **Pimoroni Pico RGB Keypad**.

Built and tested on a Raspberry Pi Pico using Pimoroni’s custom MicroPython firmware.

---

## 🚀 Getting Started

### 1. Flash the Firmware

Install Pimoroni MicroPython on your Pico:

1. Hold **BOOTSEL** while plugging in your Pico
2. Copy the Pimoroni `.uf2` firmware onto the drive
3. The Pico will reboot automatically

---

### 2. Connect with Thonny

- Open Thonny
- Select interpreter: **MicroPython (Raspberry Pi Pico)**
- Connect to the device

---

### 3. Run an Example

1. Open any `.py` file from this repo
2. Save it to the Pico (or run directly)
3. Press keys and enjoy 🎉

---

## 📁 Projects Included

### 🎨 `paint_mode.py`
Turn the keypad into a mini pixel art canvas.  
Press keys to cycle through colors and create patterns.

---

### 🌈 `rainbow_keys.py`
Each key lights up with a different color when pressed.

---

### 🎹 `color_piano.py`
Keys act like musical notes (printed to console) with colorful feedback.

---

### ⚡ `reaction_game.py`
A random key lights up — hit it as fast as you can!

---

### 🧠 `memory_game.py`
Watch the sequence, then repeat it.  
Gets harder every round!

---

### 🎲 `dice_roller.py`

---

### ✨ `fireflies.py`
A relaxing animated light pattern. Press keys to interact.

---

### 🔒 `secret_code.py`
Enter a hidden key sequence to unlock it.

---

### 💥 `speed_test.py`
How many key presses can you make in 10 seconds?

## 🧠 What You’ll Learn

These examples introduce:

- Reading button states
- Controlling RGB LEDs
- Game loops and timing
- State tracking
- Simple animations
- Event-driven programming

---

## 🧰 Requirements

- Pimoroni Pico RGB Keypad
- Raspberry Pi Pico (or Pico W)
- Pimoroni MicroPython firmware
- Thonny IDE (recommended)

No additional libraries required — everything uses built-in modules.

---

## 🛠️ Tips

- Save a file as `main.py` to run it automatically on boot
- If something freezes, unplug and reconnect the Pico
- Use `print()` in Thonny to debug behavior
